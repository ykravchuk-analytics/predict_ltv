Розрахуй фактичний LTV користувачів та побудуй предикт на рік на основі тижневих
когорт на прикріпленому семплі даних. Побудуй візуалізацію кривих предиктового та
фактичного LTVQ

Опиши обраний підхід та оціни якість побудованої моделі, запропонуй варіанти, як
можна покращити точність предикту

Завдання

Опис стовпців:
- user_id — ідентифікатор користувач
- event_timestamp — дата та час здійснення користувачем покупк
- first_purchase_time — дата та час здійснення користувачем першої покупк
- cohort_week — когорта користувача, дата початку календарного тижнv
- lifetime_weeks — тривалість життя користувача на момент конкретної події у
тижняm
- product_id — ідентифікатор підписки, купленої користувачем. Містить в собі
інформацію про вартість та тривалість підписки
